## 一、v1到v2的主要变更

### 1.1 核心发现（QVE-1结果）
- ✅ **统计学显著性极高**：四类文本在TwoNN内蕴维数上差异达到p<0.0000
- ❌ **维度阶梯顺序反转**：实际发现 D(有效) < D(重复) < D(幻觉) < D(乱码)
- ❌ **样本质量问题**：A类不是真正的重复，B类包含拒绝回答
- ❌ **缺少关键指标**：未实现关联维数D₂
- ✅ **乱码类别完美分离**：验证了假说的部分正确性

### 1.2 主要变更
| 变更项 | v1 | v2 | 原因 |
|:---|:---|:---|:---|
| 模型选择 | 1.5B Instruct | 7B Base | 避免安全机制，验证规模效应 |
| 样本构造 | 简单prompt | 标准数据集+后处理 | 确保样本质量 |
| 指标体系 | TwoNN + MLE + LZ | D₂ + TwoNN + MLE + λ + LZ | 补充核心理论指标 |
| 样本量 | 10/类 | 30-50/类 | 提高统计检验power |
| 理论框架 | 原四类映射 | 新假说+验证 | 基于实际发现调整 |

### 2.3 关键问题识别
1. **样本问题**：

## 四、更新后的实验设计（v2）

### 4.1 核心假说（v2，基于QVE-1发现）

**新假说1：结构化低维表示**
- 有效推理形成规整的低维轨迹（推理过程结构化）
- 幻觉需要编造事实，探索更多自由度（高维）
- 乱码完全随机，维度最高
- 预期维度阶梯：D(有效) ≈ D(重复) < D(幻觉) < D(乱码)

**新假说2：模型规模依赖**
- 在小模型上，有效推理不需要高维空间
- 在7B/70B大模型上，可能看到预期的阶梯或不同模式

### 4.2 实验总体架构（v2）
```
Phase 0: 远程环境搭建（新）
  ├── QVE-0v2: 远程工程流程打通
  └── 部署7B Base模型

Phase 1: 核心实验（修正版）
  ├── QVE-1v2: 四类文本维度阶梯重新验证
  │   ├── 使用7B Base模型
  │   ├── 重新设计样本构造
  │   ├── 补充关联维数D₂
  │   └── 样本量增加到30-50/类
  ├── QVE-1bv2: 距离度量选择验证
  └── QVE-1cv2: PCA降维验证

Phase 2: 辅助验证
  ├── QVE-2: D₂估计对序列长度的敏感性
  ├── QVE-3: 跨层ID曲线分析
  ├── QVE-4: 李雅普诺夫指数快速探针
  ├── QVE-5: LZ压缩率冗余性检查
  └── QVE-6: 跨模型交叉验证

Phase 3: 探索性实验
  └── QVE-7: CGR映射视觉直觉验证
```

**预计耗时**：2-4小时

### 5.2 Phase 1: 核心实验（修正版）

#### 5.2.1 QVE-1v2: 四类文本维度阶梯重新验证（生死实验v2）

**要排除的风险**：R1, R2, R9, R10

**实验设计（v2）**：

##### 数据：重新构造四类样本
```
A类 (重复/死循环) - 30条:
  模型: Qwen2.5-7B-Base (非Instruct!)
  方法: Temperature=0.0001, 无repetition_penalty
  Prompt: 超简单prompt ("aaaaa", "11111", 或重复单个词)
  后处理: 自动检测是否为真正重复/死循环，否则丢弃
  预期: 真正的重复文本
  预期D₂: <5

B类 (流畅幻觉) - 30条:
  方法1: 使用HaluEval标准幻觉数据集
  方法2: 使用TruthfulQA中模型倾向于出错的题目
  方法3: 开放式问题（避免明显虚假事实）
  后处理: 人工筛选，剔除拒绝回答
  预期: 流畅但事实错误的文本
  预期D₂: 介于有效和乱码之间

C类 (有效推理) - 30条:
  方法: GSM8K正确解答 + 自然语言解释
  Prompt: 模型擅长的推理问题
  预期: 逻辑正确的推理链
  预期D₂: 介于重复和幻觉之间，或最低

D类 (乱码) - 30条:
  方法: Temperature=5.0+, 无top-p/top-k约束
  Prompt: 任意
  预期: 语义崩坏的乱码
  预期D₂: >15
```

##### 度量：完整指标体系
```
对每条输出计算：

1. 关联维数 D₂ (Grassberger-Procaccia)
   - 核心理论指标
   - 实现: skdim.id.CorrInt() 或自实现
   - 距离: 欧氏 vs 余弦 (QVE-1b同步)

2. TwoNN 内蕴维数 (LID)
   - 计算效率高，适合对比
   - 实现: skdim.id.TwoNN()

3. MLE 内蕴维数
   - 与TwoNN对比
   - 实现: skdim.id.MLE()

4. 李雅普诺夫指数 λ (QVE-4同步)
   - 互补动力学指标

5. LZ压缩率
   - 参考基线
   - 实现: zlib.compress()
```

##### 分析方法
```
统计检验:
  - Kruskal-Wallis检验（整体差异）
  - 两两Mann-Whitney U检验
  - 计算效应量（Cohen's d）

可视化:
  - 箱线图（每个指标一张）
  - 散点图矩阵（指标间关系）
  - 层对比图（Layer vs ID）
```

##### 判定标准（v2）
```
GO条件 (任一满足即可继续):
  ✓ 四类的中位数呈现任一清晰阶梯（不管顺序！）
  ✓ 至少三类可分且统计显著
  ✓ 某一指标上四类显著分离

PIVOT条件:
  △ 仅在特定层可见分离 → 调整为逐层分析
  △ D₂不行但LID可以 → 锁定LID为主指标
  △ 顺序与1.5B一致 → 验证"结构化低维"假说
  △ 顺序反转回预期 → 验证"模型规模依赖"假说

KILL条件:
  ✗ 所有指标、所有层、四类完全混在一起
```

**预计耗时**：2-3天（含数据生成 + 维度计算）

#### 5.2.2 QVE-1bv2: 距离度量选择验证
**要排除的风险**：高维空间距离退化

**实验内容**：
对QVE-1v2的同一批数据，用三种方式计算维度：
1. 欧氏距离（L2）
2. 余弦距离（1 - cosine similarity）
3. PCA降维到50维后再用欧氏距离

**判定标准**：
- 如果三种度量给出一致趋势 → GO
- 如果仅某种度量有效 → 锁定该度量
- 如果全部无效 → 考虑更深的降维策略

**预计耗时**：与QVE-1v2同步，+2小时

---

#### 5.2.3 QVE-1cv2: PCA降维验证（新增）
**目的**：验证高维空间距离退化的影响

**实验内容**：
1. 对hidden states做PCA，保留不同比例的方差（90%, 95%, 99%）
2. 在降维后的数据上计算维度指标
3. 与原始结果对比

**预计耗时**：+2小时

### 5.3 Phase 2: 辅助验证

#### 5.3.1 QVE-2: D₂估计对序列长度的敏感性
**要排除的风险**：R4

**实验内容**：
选择一条较长的有效文本生成（512+ tokens），分别截取：
- N = 32, 64, 128, 256, 512 tokens
计算D₂和LID，画N vs. D曲线

**判定标准**：
- GO：从N=128开始，D值趋于稳定（波动<10%）
- PIVOT：需要256+才稳定 → 后续实验要求更长序列
- KILL：到512还在剧烈波动 → D₂不可靠

**预计耗时**：2-3小时

---

#### 5.3.2 QVE-3: 跨层ID曲线分析
**要排除的风险**：R6

**实验内容**：
对三类输入（自然语言、CoT、重复），画出Layer vs. ID曲线
- 分析：是否有"先升后降"模式？
- 分析：不同类型的曲线形态差异？
- 分析：d(ID)/d(Layer)（层间变化率）是否更有区分力？

**判定标准**：
- GO：至少两类展现出明显不同的模式
- INFO：纯信息收集，不构成KILL

**预计耗时**：1-2小时

---

#### 5.3.3 QVE-4: 李雅普诺夫指数快速探针
**要排除的风险**：R5

**实验内容**：
1. 选C类（有效）和A类（重复）各5条样本
2. 对prompt embedding加微小高斯噪声（ε=1e-4 ~ 1e-3）
3. 重新推理，计算轨迹分离度
4. 估算λ

**判定标准**：
- GO：有效文本的λ > 0，重复文本的λ ≤ 0
- PIVOT：λ不稳定 → 暂缓此指标
- INFO：即使λ不可用，D₂+LID已足够

**预计耗时**：半天到1天

---

#### 5.3.4 QVE-5: LZ压缩率冗余性检查
**要排除的风险**：R8

**实验内容**：
- 用QVE-1v2数据，计算每条样本的LZ压缩率
- 画D₂ vs. LZ散点图
- 计算Pearson/Spearman相关系数

**判定标准**：
- 理想：相关性中等（0.3-0.7）→ D₂提供独立信息
- 可接受：高相关但D₂在幻觉检测上更优
- 需调整：几乎完全相关 → 调整论文叙事

**预计耗时**：1小时

---

#### 5.3.5 QVE-6: 跨模型交叉验证
**要排除的风险**：R7

**实验内容**：
- 用QVE-1v2中表现最好的参数
- 在第二个模型上重复（如LLaMA-3-8B-Base）
- 仅需A/C/D三类各20条

**判定标准**：
- GO：第二个模型上同样可见分离趋势
- PIVOT：量值不同但趋势一致 → 归一化处理
- 需关注：方向完全不同 → 模型特异性

**预计耗时**：1天

### 5.4 Phase 3: 探索性实验

#### 5.4.1 QVE-7: CGR映射视觉直觉验证

### 7.2 QVE-1v2具体判定

| 场景 | 7B Base上的发现 | 判定 | 行动 |
|:---|:---|:---|:---|
| **场景1** | 顺序与1.5B一致（有效最低） | **GO** | 验证"结构化低维"假说，全速推进 |
| **场景2** | 顺序回到预期（有效中等） | **GO** | 验证"模型规模依赖"假说，全速推进 |
| **场景3** | 至少三类可分但无清晰顺序 | **PIVOT** | 分析跨层变化率或拓扑特征 |
| **场景4** | 完全混在一起不可分 | **KILL** | 放弃此方向，或转向token embedding |

### 7.3 最终判定标准（v2）

#### 全速推进（Full GO）
以下条件**全部满足**：
- [ ] QVE-1v2：四类文本显著可分（p<0.01）
- [ ] QVE-2：序列长度128+时维度估计稳定
- [ ] QVE-3：跨层曲线展示出可解释的模式
- [ ] 有合理的理论解释（不管顺序如何）

满足时的行动：**按原定Research_Roadmap执行，调整理论框架**

#### 有条件推进（Conditional GO）
以下条件**至少一个满足**：
- [ ] QVE-1v2中只有两类可分但效果量大
- [ ] 某一个指标可以，另一个不行
- [ ] 某些层可以，另一些不行

满足时的行动：
1. 缩小论文claim范围
2. 根据PIVOT方向调整实验设计
3. 重新评估NeurIPS时间线

#### 方向性调整（Major Pivot）
QVE-1v2中：
- [ ] Hidden states的D₂/LID无清晰模式
- [ ] 但token embedding空间展示出维度差异

满足时的行动：
1. 切换分析对象从hidden states到token embeddings
2. 重新设计计算流程

#### 停止（KILL）
以下条件**全部满足**：
- [ ] QVE-1v2中所有指标、所有层、所有距离度量下四类完全无法区分
